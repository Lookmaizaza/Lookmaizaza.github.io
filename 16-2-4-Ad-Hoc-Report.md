# 16.2.4 Live Lab: Build an Ad-Hoc Report

## 🎯 วัตถุประสงค์ของแล็บ (Lab Objective)
จำลองสถานการณ์จริงในโลกธุรกิจที่ต้องสร้างรายงานเฉพาะกิจ (Ad-Hoc Report) แบบเร่งด่วน เพื่อตอบคำถามหรือข้อสงสัยเฉพาะเจาะจงจากผู้บริหารหรือทีมงาน โดยไม่ต้องพึ่งพารายงานมาตรฐาน (Standardized Dashboard) 

## 🛠️ สิ่งที่ได้ลงมือทำ (What I Did)
* **วิเคราะห์โจทย์ (Requirement Analysis):** ตีความคำถามเพื่อระบุว่าต้องใช้ข้อมูลฟิลด์ (Field) หรือคอลัมน์ใดบ้างในการหาคำตอบ
* **สกัดและคัดกรองข้อมูล (Data Extraction & Filtering):** ดึงเฉพาะข้อมูลที่เกี่ยวข้องกับคำถาม และทำการฟิลเตอร์ (Filter) เอาข้อมูลหรือตัวแปรอื่นที่ไม่จำเป็นออกไป (Remove Data Noise)
* **สร้างรายงานสรุปผล (Report Generation):** นำข้อมูลที่คัดกรองแล้วมาจัดเรียง สรุปผลรวม และสร้างเป็นรายงานตารางที่ตอบโจทย์ความต้องการได้ทันที

## 🧠 สิ่งที่ได้เรียนรู้ (Detailed Learnings)
แล็บนี้เน้นหนักในเรื่องความคล่องตัวในการจัดการข้อมูล (Agility) และการตอบปัญหาอย่างตรงจุด:

* **การประเมินความต้องการข้อมูล (Identify the Needs):**
  * ได้เรียนรู้ว่าทักษะที่สำคัญพอๆ กับการใช้โปรแกรมคือ "การตั้งคำถามกลับ" เพื่อกำหนดขอบเขตของรายงานให้ชัดเจนว่าต้องการดูข้อมูลในมิติใดบ้าง
* **การจัดการกับ Data Noise:**
  * ได้ฝึกทักษะในการเลือกเฉพาะข้อมูลที่จำเป็น (Relevant Data) หากใส่ข้อมูลที่ไม่ตอบโจทย์ลงไปในรายงานฉุกเฉิน จะทำให้เกิดความสับสนและเสียเวลาในการตีความ
* **ความรวดเร็วเหนือความสวยงาม (Speed over Aesthetics):**
  * สำหรับ Ad-Hoc Report ความไวและความถูกต้องของข้อมูลสำคัญกว่าการทำกราฟิกให้สวยงาม ได้เรียนรู้วิธีสรุปข้อมูลให้เร็วที่สุดเพื่อให้ทันต่อการนำไปตัดสินใจ (Actionable insights)
* **การสื่อสารเพื่อการตัดสินใจ (Decision-Making Support - Obj 3.1):**
  * เข้าใจรูปแบบการนำเสนอที่เน้นความกระชับ (Conciseness) รายงานที่ดีต้องช่วยให้ผู้รับสารสามารถสแกนด้วยสายตาแล้วพบคำตอบได้ในเวลาไม่กี่วินาที
